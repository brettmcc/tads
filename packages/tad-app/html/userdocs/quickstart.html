Quick Start

Welcome to Tads!

Tads in a Nutshell

Tads is a fast desktop viewer for tabular data files: .parquet, .csv, .tsv, .duckdb, and .sqlite. It is a fork of Tad that adds a Stata-style command bar for filtering, sorting, and summarizing data (see The Stata Bar below). Files are imported into an embedded DuckDb database, so even very large Parquet files open quickly and scroll smoothly.

To open a file: double-click it in Explorer or Finder (the installer registers Tads for the supported file types — on Windows, right-click a file and choose Open with… > Tads to make it the default), drag it onto the Tads window, use File…Open, or launch from the command line (tad somefile.parquet) after the one-time setup below.

Everything from the original Tad is still here: pivot tables (click the pivot icon on the left to open the column selector and choose pivot, show, and sort columns), compound filters (the Filter link at the bottom of the window), and a data sources sidebar for browsing folders and database files. Those features are covered in depth in the legacy Tad documentation.

The Stata Bar

The command bar at the bottom of the window (just above the status footer) accepts a small, deterministic Stata-style command language for exploring the currently loaded dataset. It is not a general Stata interpreter: commands are schema-aware, compile to read-only SQL against the embedded DuckDb database, and show their output — and the generated SQL — in an append-only results pane above the grid.

Commands

Command names may be abbreviated as shown (sum, tab, bro, …), and options may be abbreviated where unambiguous: sum, d and tab x, m are valid. An omitted varlist means all currently visible variables.

Variables, wildcards, and if expressions

Variable names resolve like Stata's: an exact match wins, and an unambiguous prefix is accepted (tab dep can resolve to department). In a varlist, * matches any run of characters and ? matches one character, so sum price* summarizes every variable starting with price. Names containing spaces or other awkward characters are backtick-quoted: sum `gross margin`.

if expressions combine comparisons with &, |, and parentheses: ==/=, !=/~=, <, <=, >, >= against numbers, quoted strings, other columns, null, or date literals like date("2024-01-31"). Stata missing-value idioms are recognized: x == . tests missingness, x < . means non-missing, and x == "" matches both null and the empty string.

Session behavior

The command session sees the columns currently visible in the grid, so keep, drop, order, and the Columns sidebar all shape what later commands operate on. keep if and drop if accumulate a persistent row filter that later statistics combine with their own if clauses; the filter resets when you open a different dataset. A browse if filter, by contrast, changes only the grid and is not added to the session filter.

Command bar controls

  • Press Enter or click Run to execute a command.
  • Press Up/Down to navigate command history; PageUp recalls the last submitted command.
  • Press Tab after a partial variable name to complete it; repeated Tab cycles through matches.
  • Click Results or press Ctrl+` to toggle the results pane; Clear clears command history.
  • Click Break to interrupt a long-running query.

Examples

bro make price mpg if mpg > 20
sum price weight if foreign == 1
sum price*, detail
tab rep78, missing
count if price != .
keep if year >= 2020
drop temporary_*
gsort -price make
histogram price if price > 0, bin(30)

The full reference — including the expression grammar, quoting rules, output details, and safety guarantees — is in doc/stata-commands.md.
